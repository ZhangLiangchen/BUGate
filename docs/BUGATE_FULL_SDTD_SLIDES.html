② 设计哲学

五条宪法不变量(单项检查与宪法冲突时,宪法赢)

1业务含义先于实现
先确立业务上的"对/错",再谈实现细节;用户故事/验收标准/PRD 原文都是输入,转成带原文锚点的 P-xxx 命题、O-xxx oracle 后才算工件。
2命题可测·有界·或显式延后
每条重要命题要么可测、要么有明确边界、要么被显式标注延后;不留悬空的模糊主张。
3用例可回溯命题与 oracle
每条 case 回溯到 P-xxx 命题与(适用时)业务 O-xxx oracle;断言→case→命题→原文/probe 任一环断了不进代码;HTTP 200 永不是业务 oracle。
4证据分档·不升级
数据标 fact / inferred / unknown 三档;为过门把 inferred/unknown 改成 fact 被禁止——正确动作是 probe、提问或移出范围。
5绕过预期的绿≠质量
一次"绿"若绕过被测的预期行为,就不是质量证据;宽断言、适配错误、假数据都违反;xfail 变 pass、skip 被移除是真实信号,必须保留。
出处:.shared/skills/bugate/references/sdtd-constitution.md(SUT 无关的 5 条不变量,以该文件逐字为准)。"未过门禁不写代码"由物理写入门 hook 强制落地;SUT profile 只能让这些不变量更严,不能更松。

② 方法论支撑

9-Wave 不是灵感一闪 —— 站在 40 年测试研究之上

7/9 个 Wave 直接坐落在有标准、有奠基论文的经典测试方法论上;原创之处是「面向 LLM 失败模式的重新编排」,而非重新发明各项技术。地基是公认的,楼是原创的。
Wave所基于的经典方法论奠基 / 标准出处
0 PRD 健康度需求可测性 / 需求静态测试ISO/IEC/IEEE 29148 · ISTQB
1 多 AI 独立提取独立测试设计 · 规格导出测试IEEE 1028
2 引用回溯审计需求可追溯性矩阵ISO 29148 · DO-178C
4 行为 oracle测试预言问题Weyuker 1982 · Barr 等 2015
5 边界/状态机/风险边界值 · 状态迁移 · 风险驱动测试Myers 1979 · Beizer 1990 · ISO 29119
6 对抗式红队fuzzing · property-based testingMiller 1990 · Claessen & Hughes 2000
7 三层 agent 隔离独立验证与确认(IV&V)IEEE 1012
8 质量门变异测试DeMillo, Lipton & Sayward 1978
真正的原创:经典方法论解决「如何设计有效测试」;9-Wave 新增解决「当生成测试的是 LLM 时,如何防它在业务理解与缺陷发现两环节系统性失败」—— 双层架构、验证有据性≠正确性、多 AI 分歧路由、零领域知识定位,这部分既有文献没有现成答案。(Wave 3 结构化访谈基于需求获取经典实践,为压缩省略)

④ 各环节 · 1/7

Wave 1:双模型多视角需求理解

回答什么:单一 AI 读需求有一致性盲区——它漏掉的边界,自己复查一百遍还是漏。两个不同厂商模型互不沾染地独立读同一份输入,分歧本身就是风险信号。

实现

  • input pack 只含原始材料引用,不含当前 agent 的任何分析结论
  • CLI bridge 调对端请求最强推理档,输出先过 schema 校验,不过不落盘,失败件归档 cli_bridge_failures/
  • 分歧报告要求 Layer 1 吸收时生成 absorption_todo,吸收后重跑直到闭环

门禁与产物

  • 00_multiview/:双 view + divergence_report + 全程 metadata(模型/档位/耗时/prompt sha256)
  • sdtd_multiview.py check;对 Full SDTD UC 该检查被 Layer 1 语义门强制内嵌
预期效果:把「需求理解偏差」从不可见变成有产物、有归档、必须处置的显式分歧。
起源示例(Hypervise,详见附录):Claude 视角首跑被 5 项 schema 校验拒绝后重跑通过;分歧 worker 首跑因「要求吸收未吸收」被拒,吸收后闭环——两次拒绝是「门禁不是橡皮图章」的最硬证据。

④ 各环节 · 2/7

Layer 1 业务理解门 + 1A 域模型 / 1B 状态流

回答什么:需求在业务上到底说了什么?SUT 边界在哪?哪些是事实、哪些是推断、哪些没人知道?对象与状态机长什么样?
L101_business_brief
PRD Health 体检(8 维评分 + 可追溯性加成);P-xxx 命题(带原文逐字引用/锚点);O-xxx oracle(带可观测面);GAP/Q 未知项清单(高危未答阻塞);fact/inferred/unknown 三档标注。
门禁:brief 语义门 + 分档可验证性比率(≥0.80 健康 / <0.40 FAIL)
1A01a_domain_model
业务对象、属性、关系、不变量从 brief 正文升级为一等公民产物,可单独审、单独引用。
门禁:全量 pre-code 语义门(--scope pre-code)
1B01b_state_flow
状态、合法迁移、终态、异步行为、禁止迁移显式建模。
异步过渡态(如「主记录完成但明细异步落库」)是常见高危状态风险
预期效果:下游全部检查都建立在命题之上——Layer 1 写歪,后面每层都会「忠实地实现错误的模型」,所以这里是全链路最重的人工审核点。

Layer 2 可测性 / 2A 维度矩阵 / Layer 3 用例蓝图

L202_testability

  • PASS / FAIL / XFAIL / SKIP 四种信号谁也不许冒充谁
